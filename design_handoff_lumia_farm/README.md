# Handoff: LUMIA Farm — Discord Activity 게임 화면

## Overview
LUMIA Farm은 Discord 봇 **Activity(임베디드 앱)** 안에서 돌아가는 탑다운 2D 픽셀 농사 게임입니다.
스타듀밸리 / 헤이데이 / Magic Garden 류의 따뜻하고 아기자기한 타일 기반 농장 룩이 목표입니다.
이 핸드오프는 **메인 게임 월드 화면**(플레이어가 돌아다니며 작물을 심고 수확하고, 중앙 상점가를 이용하는 화면)의 디자인을 다룹니다.

플레이어는 격자(grid) 위를 **한 칸씩** 이동하며, 자기 농작지에서 작물을 심고/수확하고, 맵 중앙의 상점가에서 거래합니다. 다른 플레이어 캐릭터도 같은 월드에 함께 보입니다.

## About the Design Files
이 번들에 들어 있는 파일은 **HTML로 만든 디자인 레퍼런스(프로토타입)** 입니다. 의도한 비주얼과 동작을 보여주기 위한 것이지, 그대로 가져다 프로덕션에 넣을 코드가 아닙니다.

- 작업의 목표는 이 HTML 디자인을 **대상 코드베이스의 기존 환경에서 다시 구현**하는 것입니다.
- Discord Activity는 보통 **iframe 안의 웹 앱**(React + Canvas/Pixi.js 등)으로 만듭니다. 기존 프로젝트가 있다면 그 스택과 패턴을 따르세요. 없다면, Discord Embedded App SDK + React + Canvas(또는 PixiJS/Phaser 같은 2D 렌더러)가 자연스러운 선택입니다.
- 렌더링은 현재 **단일 `<canvas>` + 수동 픽셀 드로잉**으로 구현돼 있습니다. 프로덕션에서는 동일한 타일/스프라이트 시스템을 PixiJS나 Phaser의 타일맵으로 옮기는 것을 권장합니다(스프라이트 시트, 카메라, 컬링 등을 엔진이 처리).

## Fidelity
**High-fidelity (hifi).** 색상·타일 크기·레이아웃·인터랙션이 최종 의도에 가깝게 구현돼 있습니다. 단, 캐릭터·작물·데코는 현재 **코드로 그린 픽셀 아트(임시 에셋)** 입니다. 프로덕션에서는 같은 실루엣/색감을 유지하되 실제 **스프라이트 시트 에셋으로 교체**하는 것을 전제로 하세요. 레이아웃·색·UI 칩/HUD는 그대로 픽셀에 맞춰 재현하면 됩니다.

## Screens / Views

### 1) 메인 월드 화면 (`LUMIA Farm.dc.html`)
플레이어가 실제로 플레이하는 화면. 800×600 뷰포트 안에서 카메라가 플레이어를 따라다니며, 전체 월드는 그보다 훨씬 큽니다.

**Layout (월드 좌표계)**
- 타일 크기: **40×40 px** (`TILE = 40`). 모든 위치는 타일 격자에 스냅됩니다.
- 뷰포트: **800×600** (`VW=800, VH=600`). 카메라가 플레이어 중심을 따라가고, 월드 가장자리에서 클램프됩니다.
- 월드 크기: 가로 약 **91 타일** × 세로 **42 타일** (농작지 6열 배치 기준 자동 계산).

**농작지(plot) 배치**
- 농작지는 모두 **동일한 기본 크기**: **13×12 타일** (울타리 포함). 울타리 안 내부는 가로 **5칸 + 가운데 길 1칸 + 5칸**, 세로 **10칸**.
- 배치: 상단 행에 농작지들, 하단 행에 농작지들. **왼쪽 2열 + 오른쪽 2열**(`plotCols = [0,1,4,5]`)만 농작지로 쓰고, **가운데 2열(인덱스 2,3) 자리는 비워 상점가(마켓 광장)** 로 사용.
  - 상단 농작지: 게이트(출입구)가 아래(중앙 도로)를 향함.
  - 하단 농작지: 게이트가 위를 향함.
- 컬럼 X 좌표: `colsX[i] = 2 + i*(13+2)` → `2, 17, 32, 47, 62, 77` (타일 단위). 상단 plot Y=2, 하단 plot Y=28.

**도로/광장**
- 세로 도로: 농작지 컬럼 사이마다 **폭 2타일** (`roadsX = colsX[i]+13` → 15,30,45,60,75).
- 중앙 가로 도로(마켓 스트리트): **y=18~23** 전체 폭.
- 중앙 마켓 광장: 가운데 두 컬럼 영역(`colsX[2]`~`colsX[3]+12`, y=8~33)을 **floor(포장 바닥)** 으로 채움.

**상점(8개)** — 모두 마켓 광장에 배치. 각 상점은 5×2 타일, 천막(awning)+간판 형태.
중앙 도로 위쪽 줄(y=15)에 4개, 아래쪽 줄(y=25)에 4개. 상점 중심 X = `[35,42,49,56]`.

**타일 종류 (`cell.t`)**
- `grass` — 잔디 배경. 3가지 팔레트 변종 + 픽셀 노이즈 + 잔디 풀잎.
- `road` — 흙길(세로/중앙 도로). 베이지 + 돌 디테일.
- `floor` — 포장 바닥(상점 광장). road보다 밝은 베이지.
- `soil` — 경작 가능한 흙(밭 고랑 무늬). 작물을 심을 수 있음.
- `locked` — 미경작/잠금 땅(짙은 갈색). 확장 전 영역. 심을 수 없음.

**작물 (5종)** — 각 3성장단계(0 새싹 → 1 성장 → 2 성숙), 성숙 시 `ready=true`면 수확 가능(반짝임 + 글로우).
- carrot(당근), wheat(밀), strawberry(딸기), pumpkin(호박), star(별과일/마법작물).

**캐릭터**
- 플레이어 + 다른 플레이어들(Luna/Aria/Pico). 둥근 블롭형 픽셀 크리처 + 귀 + 눈 + 볼터치 + 이름표.
- 이동 시 hop(통통 튀는) 애니메이션, 그림자, 방향(dir: 좌/우) 반전.

**HUD (캔버스 위 DOM 오버레이)**
- **상단 바**: 좌측에 아바타 + 닉네임 + `Lv N · 농장`. 우측에 골드(🪙 G), 루나(🌾 LN) 재화 칩 + 인벤토리 버튼(🎒).
- **하단 중앙 힌트**: 상호작용 가능 시 표시. 키 캡(예: `E`) + 텍스트(예: "수확하기", "씨앗 심기", "○○ 상점 열기").
- **좌하단 컨트롤 안내**: 이동/상호작용 키 안내.

### 2) 디자인 스펙 시트 (`LUMIA Farm Design Spec.dc.html`)
개발용 참고 문서(캔버스 모드). 두 프레임:

## Interactions & Behavior

### 이동 — **한 칸씩 격자 이동 (tile-step)**
- 입력: `WASD` 또는 방향키.
- 동작: 플레이어는 **한 번에 정확히 한 타일**씩 이동합니다. 현재 타일에 도착(스냅)해 있을 때만 새 입력을 받아 다음 타깃 타일을 정하고, 그 타일 중심까지 부드럽게 보간(ease)한 뒤 멈춥니다. 키를 누르고 있으면 도착할 때마다 다음 칸으로 연속 이동합니다.
- 대각선 이동 없음(상하좌우 한 축만). 월드 경계에서 클램프.
- 구현 요지(현재 코드, `update(dt)`):
  - `player.tx, player.ty` = 목표 타일의 픽셀 좌표.
  - 도착 판정: `|x-tx|<0.5 && |y-ty|<0.5`. 도착 시 눌린 키로 `ngx/ngy = round(x/T)+dir` 계산해 새 타깃 설정.
  - 보간: `x += (tx-x)*min(1, 0.34*dt)` (대략 타일당 ~0.2초). 속도감은 이 계수로 조절.
- **카메라**: 플레이어 중심을 부드럽게 추적(`cam += (target-cam)*0.12*dt`), 월드 경계 클램프.

### 상호작용 — `E` 키 (우선순위 순)
1. 근처 상점 범위(±1.6타일) 안 → 해당 **상점 모달** 오픈(kind별 레이아웃, 아래 "상점 시스템").
2. 수확 가능한(`ready`) 작물 위 → **수확**: 작물 리셋(stage 0, 다시 성장 시작), 노란 파티클, 해당 작물 1개가 **인벤토리에 추가**(가득 차면 실패 토스트).
3. 빈 `soil` 타일 위 → **씨앗 심기**: 새 작물 생성(stage 0), 초록 파티클, 성장 타이머 시작.
4. 위 어느 것도 아니면(빈 땅/길 위) → **인벤토리 모달** 오픈. 상단 바 🎒 버튼과 동일.

### 작물 성장 시스템
- 각 작물: `{crop, stage(0~2), ready, secTotal, growLeft}`. `secTotal`은 작물별 총 성장 시간(초): 당근 240 · 밀 180 · 딸기 420 · 호박 600 · 별과일 1200.
- 성장은 3페이즈(새싹0 → 성장1 → 성숙2 → 수확가능). `update(dt)`에서 `growLeft -= dt/60`(초). 0이 되면 `stage++` 후 `growLeft = secTotal/3`, stage 2 이후엔 `ready=true`.
- 프로덕션: 성장은 **서버에서 `plantedAt` 타임스탬프 + 경과시간**으로 계산해 재접속 시 복원(클라 타이머는 표시용).

### 작물 정보 툴팁 (작물 위에 올라갔을 때)
- 플레이어가 선 타일에 작물이 있으면 머리 위에 말풍선 표시.
  - **다 자란 작물**: 작물 이모지 + 이름 + "수확 가능" 녹색 배지.
  - **자라는 중**: 작물 이모지 + 이름 + `🕒 남은 시간`(완성까지, `남은페이즈 × secTotal/3` 합산, "M분 S초" 포맷).
- 위치: 플레이어의 화면 좌표(`player - cam`) 기준 위쪽 52px, `translateX(-50%)`. 아래쪽 삼각 포인터.

### 상점 시스템 (8개, 5가지 레이아웃)
`kind` → `layout`: seed·petbuy = `buy` / sell·petsell = `sell` / exch = `exch` / upgrade·hire = `upg` / inventory = `inv` / storage = `store`.
- **buy**(씨앗·펫): 2열 카드 그리드(이모지·이름·설명·가격·구매). 잔액 부족 시 실패 토스트.
- **sell**(작물·펫): 보유 목록 + 1개/모두 판매 버튼 + 하단 "전체 예상 수익" 바 + 전체 판매.
- **exch**(환전소): 골드↔루나 **양방향**. ⇄ 스왑 버튼으로 방향 전환, 슬라이더+최대 버튼으로 수량, 환율 **10 G = 1 LN**.
- **upg**(업그레이드·고용): 행 리스트 + 레벨 배지(`Lv n/max`) + 강화/고용 버튼.
- **inv**(인벤토리): 10열 슬롯 그리드. 1~10번 슬롯이 하단 핫바와 연동.
- **store**(보관함): 좌(인벤)·우(창고) 2분할. 슬롯 **클릭=1개, Shift+클릭=묶음 전부** 이동(deposit/withdraw 양방향).
- 모달은 캔버스 위 DOM 오버레이. `✕` 또는 `Esc`로 닫힘, 열려 있는 동안 캐릭터 이동/입력 정지. 액션 결과는 하단 토스트(성공 녹색 / 실패 빨강).

### 인벤토리 · 보관함 · 핫바
- **인벤토리(`inv`)/보관함(`sto`)**: 각각 길이 30의 **순서 있는 슬롯 배열**. 슬롯 = `{key, count}` 또는 `null`. 같은 종류는 한 슬롯에 스택.
- **핫바**: 항상 보이는 마인크래프트식 하단 10칸 = 인벤토리 슬롯 0~9. `1`~`9`,`0` 키로 선택(선택 칸 강조 + 확대). 번호 라벨·수량 표시.
- 이동 헬퍼: `addItem(arr,key,n)`(스택 후 빈 칸), `transfer(dir,idx,amount)`(amount=1 또는 'all'), `removeKey/countKey`(판매·집계용).

### 애니메이션
- 작물: 성숙(`ready`) 시 위아래 bob + 글로우 + 간헐적 스파클.
- 데코: 꽃 흔들림, 마법 크리스탈 글로우 펄스.
- 캐릭터: 이동 시 hop, 정지 시 미세한 idle bob.
- 파티클: 수확/심기 버스트(중력 적용, 페이드아웃).
- HUD 힌트: 등장 시 pop 스케일, 키 캡 idle 바운스.

## State Management
런타임 상태(현재 단일 컴포넌트 보관):
- **플레이어**: `{x, y, tx, ty, dir, anim, moving}` (픽셀 좌표 + 타깃 타일 + 방향/애니메이션).
- **재화/프로필**: `gold`, `luna`, `farmLevel`, `name` (HUD 표시 + 거래/판매 시 변동).
- **인벤토리**: `inv[30]`(들고 다니는 아이템, 슬롯 0~9 = 핫바), `sto[30]`(보관함), `sel`(선택된 핫바 슬롯). 슬롯 = `{key, count}|null`.
- **월드**: `grid[H][W]`(타일 종류/변종), `crops[]`(성장 타이머 포함), `fences[]`, `decor[]`, `buildings[]`, `others[]`.
- **카메라**: `cam {x,y}`.
- **UI**: `shopKind`(열린 상점/모달), `showHint/hintText/hintKey`, `cropTip`(작물 툴팁), `exchGold/exchDir`(환전), `toast`(피드백).
- **파티클**: `particles[]`.

프로덕션 권장:
- 월드/소유 농작지/작물 성장 상태는 **서버 권위(authoritative)** 로 두고, 클라이언트는 보간 렌더링.
- 멀티플레이어 위치/상호작용은 Discord Activity의 네트워킹 레이어(또는 자체 WS 서버)로 동기화.
- 작물 성장은 타임스탬프 기반(심은 시각 + 성장 시간)으로 계산해 재접속 시 복원.
- 인벤토리/보관함/재화는 서버 권위로 두고 거래·이동을 트랜잭션으로 처리(클라는 낙관적 업데이트 + 롤백).

### 읽는 순서 (LUMIA Farm.dc.html 안에서)
1. `generateWorld()` — 그리드/농작지/도로/광장/상점/데코/플레이어 배치 규칙.
2. `bakeWorld()` — 타일·울타리를 오프스크린 캔버스에 한 번에 그려 캐싱.
3. `update(dt)` — **격자 한 칸 이동** 로직 + 카메라 + 파티클 + 힌트 감지.
4. `interact()` — 수확/심기/상점.
5. `draw()` + `drawCrops/drawDecor/drawBuildings/drawPlayers/drawParticles` — 렌더링.
6. 템플릿(HTML) 하단 — HUD 오버레이(상단 바/힌트/컨트롤 안내).
