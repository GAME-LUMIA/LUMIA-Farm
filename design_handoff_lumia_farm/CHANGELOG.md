## 1) 작물 시스템: 5종 → **30종 / 6티어** (`crops.js`, `buildCropInfo()`)

기존 `CROPINFO`(당근·밀·딸기·호박·별과일 하드코딩)는 **삭제**되고, `buildCropInfo()`가 `LumiaCrops.CROPS`로부터 런타임 생성합니다.

**티어 (6단계)** — 티어가 높을수록 최초 성장이 길고 값비쌈:

| 티어 | 이름 | 최초 성장 | 재성장(다회수확) | 씨앗가/판매가(코드 기준) |
|---|---|---|---|---|
| T1 | 새싹 작물 | 5~20분 | 없음(단일수확) | 10 / 18 |
| T2 | 텃밭 작물 | 30분~1.5시간 | 없음 | 25 / 45 |
| T3 | 과실 작물 | 2~6시간 | 없음 | 60 / 110 |
| T4 | 덩굴·나무 | 8~12시간 | 2~4시간 | 140 / 260 |
| T5 | 농원 작물 | 16~24시간 | 6~10시간 | 300 / 520 |
| T6 | 과수원 작물 | 30~42시간 | 14~24시간 | 600 / 1050 |

- 가격은 `tierPrice` 맵에서 티어별로 부여. `goldenapple`(황금사과)만 예외로 **루나(luna) 통화** seed 45 / sell 90.
- 성장/재성장 시간 문자열은 `parseSpec()`으로 초로 변환(`"1시간 30분"` → 5400s). `"N시간"`, `"N분"` 파싱.

**30종 전체 목록** (id · 이름 · 티어 · 최초성장 · 재성장):

T2: tomato 토마토 30분 · corn 옥수수 45분 · cucumber 오이 1시간 · broccoli 브로콜리 1시간15분 · pumpkin 호박 1시간30분  (단일)

> 정확한 데이터·색 팔레트는 `crops.js`의 `CROPS` 배열을 단일 출처(source of truth)로 삼으세요.

### 수확 방식 — 단일 vs 다회(재성장) — `harvestReset(c)`
- **단일 수확 (regrow=null, T1~T3)**: 수확하면 `crops[]`에서 제거 → **빈 흙(soil)** 으로 복귀. 다시 심어야 함.
- **다회 수확 (regrow 있음, T4~T6)**: 수확해도 작물은 남고 `ready=false; stage=2; growLeft=regrowSecs`로 리셋 → **재성장 시간 뒤 다시 열매**. **삽으로 캐낼 때까지** 무한 반복.
- 심기(`interact()` 플랜트 분기): 선택된 핫바 씨앗 종류로 심음(없으면 carrot). `secTotal=info.secs`, `growLeft=info.secs/3`.

### 삽으로 캐기 — **`X` 키** (`digUp()`)
- 플레이어가 선 타일의 작물을 **제거**하고 빈 흙으로 되돌림(재성장 작물을 걷어낼 때 사용). 갈색 파티클 버스트 + 토스트.
- 하단 컨트롤 힌트에 `X 삽으로 캐기` 추가.

### 성장 데모 가속
- `GROW_SCALE = 40` — 데모에서 실제 성장 시간을 40배 압축해 관찰 가능하게 함(`update()`에서 `gs = ds*GROW_SCALE`). **프로덕션에서는 제거**하고 서버 타임스탬프 기반으로 계산.

## 2) 픽셀 아트 아이콘 — 이모지 → 코드 스프라이트

`crops.js`가 세 종류의 캔버스 렌더러를 제공:
- **`drawCrop(ctx, id, cx, groundY, scale, opts)`** — 밭에 심긴 **전체 식물**(줄기·나무·덤불 포함). 바람 스웨이, 성장 단계별 스케일.
- **`drawFruit(ctx, id, cx, cy, scale)`** — **수확물(열매 그 자체)**. 토마토는 동그란 토마토, 옥수수는 자루, 포도/블루베리는 열매 송이, 나무 과일은 과일만. `FRUIT_OF` 맵이 작물→열매 페인터를 매핑.
- 씨앗 봉투 아이콘 — `LUMIA Farm.dc.html`의 `renderSeedIcon(c)`가 크래프트지 봉투 + 작물색 창 + 씨앗 알갱이로 생성.

게임 내 캐싱: `makeIcons()`가 작물별 `iconCrop`(=drawFruit dataURL, 인벤/핫바/판매/툴팁용)과 `iconSeed`(씨앗 상점용) PNG dataURL을 생성해 `CROPINFO[id]`에 저장.
- **중요 구현 노트**: 아이콘은 `<img src="{{hole}}">`가 아니라 `imgEl(src,size)` 헬퍼로 **완성된 React 엘리먼트**(`{{ iconEl }}`)로 렌더. (스트리밍 중 잘못된 이미지 요청 방지.)
- 인벤토리/핫바/보관함/상점 buy·sell/작물 툴팁 모두 이 픽셀 아이콘 사용. 펫은 `pets.js`의 `drawPet`으로 아바타 PNG 생성(`renderPetIcon(id)`, 캐시).

## 3) 동물 먹이 & 배고픔 시스템

### 먹이 스펙 (`crops.js` `FEED`) — 등급별 회복량이 다름
`hunger[]`·`satiety[]` 배열 인덱스 = **[커먼, 레어, 에픽, 레전더리]**. 값 0이면 그 등급엔 효과 없음(고급 펫일수록 고급 먹이 필요).

| 작물 | 배고픔 회복 [C,R,E,L] | 포만감(초) [C,R,E,L] |
|---|---|---|
| carrot 당근 | 80/20/0/0 | 10/5/0/0 |
| lettuce 상추 | 80/20/0/0 | 10/5/0/0 |
| tomato 토마토 | 100/40/0/0 | 20/10/0/0 |
| corn 옥수수 | 100/40/0/0 | 20/10/0/0 |
| watermelon 수박 | 100/80/30/0 | 30/20/5/0 |
| blueberry 블루베리 | 100/80/30/0 | 30/20/5/0 |
| grape 포도 | 100/80/60/30 | 40/30/10/5 |
| banana 바나나 | 100/80/60/50 | 50/40/30/20 |
| apple 사과 | 100/100/100/100 | 80/70/60/50 |

- **배고픔 회복**: 먹였을 때 채워지는 배고픔 지수(최대 100).
- **포만감**: 먹인 직후 배고픔이 **닳지 않는 유예 시간(초)**. 이 시간이 지나야 다시 감소 시작.

### 배고픔 소모 속도 (`crops.js` `HUNGER`, `GRADE_DRAIN`)
배고픔 100 → 0 까지 걸리는 시간. 등급이 높을수록 오래 버팀:

| 등급 | key | 소진 시간 | 초 |
|---|---|---|---|
| 커먼 | Common | 30분 | 1800 |
| 레어 | Rare | 1시간 | 3600 |
| 에픽 | Epic | 2시간 | 7200 |
| 레전더리 | Legendary | 3시간 | 10800 |

- 소모 로직(`update()` 내 펫 루프): `satietyLeft>0`이면 그 시간부터 차감(배고픔 유지), 아니면 `hunger -= ds*(100/drainSecs)`. `drainSecs = GRADE_DRAIN[grade]/HUNGER_DEMO`.
- `HUNGER_DEMO = 40` — 데모 가속 배수. **프로덕션 제거** 후 실시간/서버 계산으로.
- 배고픔 0 → 펫 `starving=true`: 일(능력) 멈추고 HUD에 `😢 배고파요 · 먹이 필요`, 아이콘 그레이스케일.

### 먹이주기 UI — **선택식 피커** (`feedListFor`, `feedPetWith`)
- 각 펫 카드 하단에 **가로 전체폭 "먹이주기" 버튼**(이모지 없음). 클릭하면 그 펫에게 줄 수 있는 **먹이 목록 팝업**이 옆에 뜸.
- 목록 각 행: 작물 아이콘 + 이름 + 보유수량, 그 펫 등급 기준 **회복량(+배고픔)·포만감 초**. 효과 있는 먹이가 초록으로 위에 정렬, 효과 없는 먹이는 흐리게 "이 등급엔 효과 없음"(클릭 불가).
- 클릭 시 인벤토리에서 1개 소모 → 배고픔 += fill, `satietyLeft = sat`, 팝업 닫힘. 이벤트 로그 발생(아래 6번).
- 인벤에 먹이 없으면 안내 문구.

## 프로덕션 구현 체크리스트 (델타)
1. 작물 데이터는 `crops.js`의 `CROPS/TIERS/FEED/HUNGER`를 **서버 설정(config/DB)** 으로 이관. 성장·재성장은 **`plantedAt`/`lastHarvestAt` 타임스탬프** 기반으로 서버 계산(클라 타이머는 표시용). `GROW_SCALE`·`HUNGER_DEMO` 데모 가속 제거.
2. 단일/다회 수확 분기(`harvestReset`)와 **삽(X) 제거** 트랜잭션을 서버 권위로.
3. 배고픔은 서버에서 등급별 실시간 감소 + 포만감 유예로 계산. 먹이 소비·회복도 트랜잭션.
4. 펫 별명·인벤토리 순서(핫바 매핑 포함)를 서버 유저 상태에 저장.
5. 이벤트 로그는 서버 이벤트 스트림으로 대체 가능(펫 능력 발동을 서버가 판정).
6. 픽셀 아트(작물 3렌더러 + 펫 아바타)는 임시 코드 아트 → **스프라이트 시트 에셋으로 교체**(같은 실루엣/색 유지).
